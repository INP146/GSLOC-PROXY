# 安全与支持政策

<p><a href="../SECURITY.md">English</a> | <b>简体中文</b></p>

本文档说明 GSLOC-PROXY 的安全研究范围、维护者支持边界、问题报告方式和使用者合规责任。它用于帮助贡献者和使用者判断哪些问题适合公开讨论、哪些场景不会获得项目支持。

## 项目范围

GSLOC-PROXY 只面向自有设备、自有应用、授权账号和受控实验网络中的安全研究、防御测试和移动应用位置风控鲁棒性验证。

本项目是网络层定位完整性测试代理，不是公共代理服务、通用 HTTPS MITM 工具、反检测工具或第三方平台策略规避工具。

## 不提供协助的内容

维护者不会为以下场景提供 issue 回复、调试支持、功能实现、配置建议、部署协助或其它任何形式帮助：

- 绕过打卡、游戏、金融、配送平台、地区限制、反作弊或平台风控；
- 针对第三方应用、第三方账号、第三方设备或未授权网络环境的测试；
- 隐藏代理、规避检测、模拟更真实轨迹、对齐 IP 与定位、绕过风控阈值；
- 将本项目作为公网代理、商业代理、共享代理或通用 MITM 服务；
- 收集、保存、分析或分享他人设备、账号、网络、定位上下文或业务数据；
- 任何违法、未授权、侵害第三方权益或明显偏离防御性测试目的的使用。

相关 issue、discussion 或 pull request 可能会被关闭、删除或锁定，维护者不承诺解释或继续沟通。

## 可接受的安全研究问题

欢迎提交与以下内容相关的问题或改进：

- 授权实验环境中的复现性、稳定性和安全边界；
- host/path allowlist、fail-closed 行为、日志脱敏和本地默认监听；
- 管理 API 的访问控制、最小权限和本地部署安全性；
- 防御性检测、位置可信度评分和风控鲁棒性建议；
- 文档中可能导致误用或误解的表述；
- 项目自身的安全缺陷。

提交问题时，请避免包含真实账号、设备标识、BSSID/MAC、抓包原文、私有 CA、密钥、token、服务器地址、完整日志或任何第三方平台数据。如需说明日志相关问题，请先脱敏并只保留最小必要片段。

## 报告安全问题

如果你发现项目本身的安全问题，请优先在 GitHub 仓库的 Security 页面使用私有安全报告功能，或通过维护者在仓库资料中公开的私下渠道联系。报告中请尽量提供最小可复现信息，不要附带敏感抓包、私钥、真实设备标识、完整日志或第三方业务数据。
